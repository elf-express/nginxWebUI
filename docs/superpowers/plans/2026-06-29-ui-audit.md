## 設計準則(Design Baseline)

User 明確指定:

- **基準 viewport**:**1280px** 寬
- **內容區域(content area)**:**1080px** 寬(扣掉 sidebar 約 200px)

- **modal 寬度**:應對齊 content area 寬度(否則蓋住 sidebar 或跑出邊界)

**Critical findings:**

1. **server modal 1346** > 1080 content area → 在 1280 viewport `x = -33`,跑出畫面
2. **upstream modal 1200** > 1080 → 在 1280 viewport 緊貼或超出
3. **無一個用 % 或 calc()** → 完全沒 RWD
4. **無一致設計準則** → 32+ 個不同寬度,看起來像隨手填值

**建議統一策略**(等 user 圈選):

- **A. 主要編輯 modal**(server/http/stream/upstream)→ `[main.width + 'px', main.height + 'px']` 動態抓 `.layui-body` rect,完全等同 content area
- **B. 中型編輯 modal**(cert/denyAllow/template)→ 固定比例(例:60% × 70%)
- **C. 小型確認/選擇 modal**(select_lang / asn 加 IP / admin 重設密碼)→ 維持小 fixed pixel(但用 vw + 上限,例:`min(450px, 90vw)`)
